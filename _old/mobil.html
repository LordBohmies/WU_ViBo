Mobil Webb

Fram till ungefär 2011-2012 så var det vanligaste sättet att surfa på en vanlig dator (desktop och laptop). Skärmstorlek och upplösning har gradvist ökats och idag är 24" den vanligaste skärmstorleken med en skiftande upplösning.

Utmaningar

Ett indirekt resultat av en exploderande marknad för just mobil surfning har lett till ett väl utbyggt 3G-nät och ett 4G-nät under ständig expansion. Tillsammans med förmånliga mobilabonnemang, där surf kan vara begränsat men även obegränsat, så ställdes då stora krav om omställning för utvecklarna. En dålig sida förbrukar användarens surfmängd helt i onödan.

Formfaktorn

Mobilerna är idag näst intill lika kraftfulla som en modern laptop, visst skärmen är liten men tack vare en kraftfull GPU så har de lika hög upplösning som stationära skärmar. De mobila enheterna har oftast endast ett mjukvarutangentbord som oftast skymmer en stor del av skärmen när de används.

Opålitlig bandbredd

Både 3G-nätet och 4G-nätet är väl utbyggda i storstadområden och 3G-nätet är även väl utbyggt ut med landets större vägar. Den största utmaningen här är att masterna belastas i olika mängd och är masten överbelastad så sjunker bandbredden drastiskt. Även om det idag är ovanligt, i alla fall söder om Mora, så kan masten vara placerad långt ifrån dig, vilket även det resulterar i en låg bandbredd. Även vid snabb förflyttning så kan det förekomma att kontakten tappas helt i någon sekund. Notera dock att man i många storstäder och på tåg så har man installerat publika Wifi-nätverk som användaren kan använda i viss mån. Kom ihåg att dessa nät är just publika och är därför under ständigt hot från inkräktare.

Svårare inmatning

Mjukvarutangentbord kan vara väldigt små och göra det svårt för inmatning i jämförelse med ett större tangentbord. Det är även viktigt att dina interagerbara element är tillgängliga för användaren när de använder en mobil enhet, särskilt om de bara har exempelvis en hand tillgänglig.

Olika enheter

iOS och Android, de två största operativsystem på mobilmarknaden. Även om de upptar 99.6% av marknaden så är de långt ifrån de enda. Visst det finns några Operativsystem att ta hänsyn till, men varje operativsystem finns i flera olika versioner. iOS finns har ett 10-tal. Android finns som det närmare 20, 5 aktuella, versioner av. Viktigt att komma ihåg att Android mjukvara och gränssnitt modifieras av varje mobiltillverkare och ibland även operatör för att passa deras marknad.

Nya tekniker

Mobila enheter kommer även med inbyggda funktioner som man tidigare inte ens tänkt tanken på att använda på en webbsida. Men inbyggd GPS, rörelsesensor (accelerometer), kamera, mikrofon, vibrator etc. är tekniker som öppnat upp måna möjligheter. Nya sätt att agera mot webbplatser, exempelvis luta mobilen i spel eller information som baseras på din geografiska position.

Kompetens

Om vi skulle gå in för mycket på dessa tekniker så skulle nivån bli för svår. Men tekniken är allt för viktig för att strunta helt i och vill du göra ett seriöst projekt så är det viktigt att du tar till dig all kunskap och lite till.

Viewport

Exempel på när man som utvecklare har använt <meta name="viewport" /> på fel sätt.

  • När man besöker en webbsida och endast halva skärmen är använd. Vilket ofta beror på att utvecklaren bara anpassat sidan efter sin egna mobila enhet.
  • När texten blir för liten för att kunna läsas och funktionen för att zooma in är borttagen.

Ovan nämnda problem kan undvikas om man som utvecklare tar sig tiden att läsa dokumentation för varje webbläsare som används.
Med <meta name="viewport" /> kan du styra sidornas bredd, höjd och zoom. Exempelvis om du i din CSS sätter bredden till 80% så kommer frågan 80% av vadå att uppstå. Med följande viewport så blir det garanterat 80% av skärmen oavsett dess bredd eller pixeldensitet.

<meta name=" viewport" content=" width=device-width, initial-scale=1.0" >

Media queries

@media, användes från början för att skilja mellan olika typer av enheter. I och med introduktionen av den mobila webben så har det tillkommit en står mängd mediafrågor (media queries). Allt för att urskilja storlek, upplösning, formfaktor och liknande. Med hjälp av dessa kan du bygga webbplatser som är anpassade efter flera brytpunkter. Exempelvis skifta mellan 3-/ och 1-spaltslayout.

@media screen and (max-width: 800px) {
  /* CSS för smala fönster/skärmar */
}
@media screen and (min-width: 801px) and (max-width: 1200px) {
  /* CSS för mellanbreda fönster/skärmar */
}
@media screen and (min-width: 1201px) {
  /* CSS för breda fönster/skärmar */
}

Använd brytpunkter efter vad din design kräven, utgå inte från enstaka produkter på marknaden. Tips är dock att kolla in Bootstrap.
Mobiler och surfplattor körs i princip alltid i helskärmsläge.
Fler exempel på mediafrågor:

@media screen and (max-width: 800px) {
  /* Bara webbläsare som förstår media queries.
  Lägg märke till ordet only */
}
@media screen and (min-aspect-ratio: 1/1) {
  /* Fönstret är minst lika brett som det är högt */
}
@media screen and (device-aspect-ratio: 1/1) {
  /* Enhetens skärm är minst lika bred som den är hög */
}
@media print and (min-resolution: 300dpi) {
  /* En skrivare med minst 300 punkter per tum i upplösning */
}
@media screen and (orientation: portrait) {
  /* Enheten hålls på höjden */
}
@media screen and (orientation: landscape) {
  /* Enheten hålls på bredden */
}

Events

Nya händelser som tillkom specifikt för det mobila behovet:

  • Touch-händelser.
  • Offline/online - när man tappar eller återfår förbindelsen med nätet.
  • Luta och vrida på enheten.
  • Telefonrelaterade händelser - exempelvis för att pausa ett spel och skicka meddelande om det till medspelare runtom i världen när telefonen ringer.

Geolocation

Tack vare att de flesta mobiler idag kommer med GPS så går de att spåra med Geolocation. Den kan även använda sig utav IP-adresser och nätverk i närheten för att lokalisera dig om du av någon anledning inte har GPS igång. Utnyttjas ofta av företag för att rikta reklam.
